# screens

A pasta `screens` contém os componentes de tela do aplicativo, cada um projetado para gerenciar uma parte específica da interface do usuário. As telas são responsáveis por renderizar a interface e interagir com o usuário, utilizando componentes e hooks para gerenciar o estado e a lógica do aplicativo.

## Estrutura da Pasta

A estrutura da pasta `screens` é a seguinte:

- **aguaConsumida**: Tela para registrar e visualizar a quantidade de água consumida.
- **cadastrarAlimento**: Tela para cadastrar novos alimentos.
- **cadastrarDieta**: Tela para criar novas dietas.
- **cadastro**: Tela para o registro de novos usuários.
- **dietasPersonalizadas**: Tela para visualizar e gerenciar dietas personalizadas.
- **dietasPredefinidas**: Tela para acessar dietas predefinidas.
- **editProfile**: Tela para editar o perfil do usuário.
- **FAQs**: Tela que apresenta perguntas frequentes.
- **home**: Tela inicial do aplicativo.
- **listAlimentos**: Tela para listar todos os alimentos disponíveis.
- **login**: Tela de login do usuário.
- **profile**: Tela que exibe as informações do perfil do usuário.
- **questionario**: Tela para responder a questionários.
- **selecao**: Tela para seleção de opções.
- **userAlimentos**: Tela que mostra os alimentos do usuário.
- **userAlimentosConsumidos**: Tela que exibe os alimentos consumidos pelo usuário.
- **userDietaDiaria**: Tela que mostra a dieta diária do usuário.
- **userDietas**: Tela para visualizar as dietas do usuário.

Cada uma dessas telas é implementada em um arquivo separado, geralmente com um arquivo de estilo correspondente para gerenciar a aparência visual.
